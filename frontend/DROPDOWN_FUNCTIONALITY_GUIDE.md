# Руководство по функционалу dropdown'ов

## Что теперь работает:

### 1. 🔄 **Кнопка сортировки (иконка с двумя стрелками)**
- Переключает направление сортировки между `asc` и `desc`
- Меняет цвет кнопки (синий/фиолетовый)
- Поворачивает SVG иконку

### 2. 📊 **Dropdown "Show" (количество элементов)**
- **Варианты**: 20, 30, 50 элементов на странице
- **ID**: `#dropdownSort2`
- **Поведение**: При выборе нового значения сбрасывает на первую страницу

### 3. 🎯 **Dropdown "Sort by" (критерий сортировки)**
- **Варианты**:
  - Most Viewed (most_viewed)
  - Recently search (recently_searched)
  - Most popular (most_popular)
  - Top rated (top_rated)
- **ID**: `#dropdownSort`
- **Поведение**: При выборе нового критерия сбрасывает на первую страницу

## API Параметры:

Теперь все запросы к API включают:
```
GET /cars?page=1&limit=20&sort=most_viewed&direction=desc
```

### Параметры:
- `page`: номер страницы
- `limit`: количество элементов на странице (20, 30, 50)
- `sort`: критерий сортировки (most_viewed, recently_searched, most_popular, top_rated)
- `direction`: направление сортировки (asc, desc)

## Как тестировать:

### 1. **Перезапустите API сервер:**
```bash
cd api
python main.py
```

### 2. **Откройте страницу** `dist/index.html`

### 3. **Тестируйте dropdown'ы:**

#### Dropdown "Show":
1. Нажмите на "Show 20"
2. Выберите "30" или "50"
3. Должно загрузиться новое количество автомобилей
4. В консоли: `Limit changed to: 30`

#### Dropdown "Sort by":
1. Нажмите на "Sort by: Most Viewed"
2. Выберите другой критерий (например, "Most popular")
3. Должны загрузиться данные с новым критерием
4. В консоли: `Sort changed to: most_popular`

#### Кнопка сортировки:
1. Нажмите на иконку с двумя стрелками
2. Кнопка должна изменить цвет
3. В консоли: `Направление сортировки изменено на: asc`

## Ожидаемое поведение:

### При изменении "Show":
- ✅ Количество автомобилей на странице изменится
- ✅ Пагинация пересчитается
- ✅ Сброс на первую страницу

### При изменении "Sort by":
- ✅ Порядок автомобилей может измениться (если API поддерживает разные критерии)
- ✅ Сброс на первую страницу
- ✅ Обновление текста в dropdown'е

### При клике на кнопку сортировки:
- ✅ Направление сортировки переключится
- ✅ Визуальное изменение кнопки
- ✅ Перезагрузка данных

## Отладка:

### В консоли браузера должны появляться сообщения:
```
Loading cars data, page 1, limit: 20, sort: most_viewed, direction: desc...
Limit changed to: 30
Sort changed to: most_popular
Направление сортировки изменено на: asc
```

### Проверка API запросов:
```bash
# Тест с разными параметрами
curl "http://localhost:8000/cars?page=1&limit=30&sort=most_popular&direction=asc"
curl "http://localhost:8000/cars?page=1&limit=50&sort=top_rated&direction=desc"
```

## Примечания:

1. **Сейчас все критерии сортировки работают одинаково** (по ID), так как в базе данных нет полей для "популярности", "рейтинга" и т.д.

2. **Для реальной работы разных критериев** нужно:
   - Добавить поля в базу данных (views, rating, popularity)
   - Обновить логику сортировки в API

3. **Dropdown'ы полностью функциональны** и готовы к расширению функционала
